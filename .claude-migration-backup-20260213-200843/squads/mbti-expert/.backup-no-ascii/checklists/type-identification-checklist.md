
# Type Identification Checklist

Validation checklist for type identification process.


---

## Pre-Assessment
- [ ] User understands this is not a certified MBTI assessment
- [ ] User is answering based on natural preferences, not situational behavior
- [ ] All 4 dimensions have been assessed


## Dimension Validation
- [ ] **E/I assessed:** At least 2 scenarios presented; clear preference identified
- [ ] **S/N assessed:** At least 2 scenarios presented; clear preference identified
- [ ] **T/F assessed:** At least 2 scenarios presented; clear preference identified
- [ ] **J/P assessed:** At least 2 scenarios presented; clear preference identified
- [ ] **Identity (A/T):** Stress resilience and self-confidence assessed


## Result Quality
- [ ] Type code is valid (one of 16 types)
- [ ] Role group correctly identified
- [ ] Strategy correctly matched to Identity variant
- [ ] Brief profile resonates with user
- [ ] User had opportunity to confirm or adjust result


## Post-Assessment
- [ ] Offered next steps (profile, growth, career, compatibility)
- [ ] Mentioned that formal MBTI assessment is recommended for validation
- [ ] Clarified that types describe preferences, not limitations
