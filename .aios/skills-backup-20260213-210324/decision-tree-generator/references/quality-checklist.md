# Decision Tree Quality Checklist

Verify ALL items before delivering the final documentation.

## Completeness

- [ ] All decision types are defined with clear definitions
- [ ] Each type has "when to use" criteria (minimum 3 conditions)
- [ ] Each type has at least 2-3 concrete examples
- [ ] Mermaid flowchart is included and renders correctly
- [ ] All criteria are documented with example scenarios
- [ ] Comparison matrix covers all types
- [ ] Quick reference checklist is actionable
- [ ] Cost-benefit analysis included (when applicable)
- [ ] Hybrid strategies documented (when applicable)
- [ ] Common mistakes section included

## Clarity

- [ ] A newcomer could follow the decision tree without prior knowledge
- [ ] No ambiguous language ("usually", "sometimes", "maybe")
- [ ] Each path leads to a clear, named outcome
- [ ] Examples are specific and concrete, not generic
- [ ] Decision questions are phrased as clear yes/no or multi-choice

## Accuracy

- [ ] Flowchart matches the written criteria exactly
- [ ] No contradictions between sections
- [ ] Edge cases are addressed
- [ ] All flowchart paths terminate at a defined outcome
- [ ] Criteria hierarchy is logical (primary -> secondary -> tertiary)

## Usability

- [ ] Document can be used as standalone reference during decision-making
- [ ] Checklist format enables quick decisions (under 2 minutes)
- [ ] Version history is present for tracking changes
- [ ] Related documents are linked when applicable
- [ ] Mermaid diagram uses distinct colors for each outcome type
