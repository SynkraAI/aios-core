# E5.1 — Redesign Mobile-First da Home Page com Conteudo Rico

## Story

**Como** visitante acessando pelo celular,
**quero** uma home page responsiva, com conteudo rico e navegacao clara,
**para** entender o programa Ciclo das Estacoes, conhecer as facilitadoras, ver a programacao tipica e me inscrever facilmente.

## Description

Redesign completo da home page (`ciclo-app/apps/web/app/page.tsx`) seguindo o wireframe mobile-first v3 e a pesquisa de referencias (basetriade.com, ecoin, Renascenca). A home atual tem problemas criticos: mandala amontoada em telas pequenas, hero desperdicando 80vh, conteudo generico sem as informacoes reais do programa, ausencia total de navbar, e apenas 3 value props genericas em vez das 6 reais.

O redesign adiciona 7 novas secoes de conteudo rico, corrige o CSS responsivo da mandala, cria uma navbar sticky mobile, e reorganiza a hierarquia de 12 secoes conforme prioridades P0-P1.

**Fontes de verdade:**
- Wireframe: `docs/design/wireframe-home-mobile-first-v3.md`
- Pesquisa: `docs/research/2026-03-07-redesign-mobile-conteudo.md`
- Visual Spec: `docs/design/visual-spec-v2.md` (componentes base)
- PRD: secao 5 (Home), NFR-01 (Performance), NFR-03 (Mobile-first)

## Dev Notes

### Arquitetura de Dados
- Todos os conteudos ricos sao constantes TypeScript (D-3 do wireframe)
- Pattern: exportar arrays/objetos tipados, page.tsx importa e renderiza
- Override futuro via `getSiteContents()` (FR-01 do PRD — admin editavel)
- [Source: wireframe-home-mobile-first-v3.md#D-3]

### Paleta de Cores — Conflito Resolvido
- `#213B2C` (verde-floresta) → navbar, footer, badges identitarios (nova var `--base-forest`)
- `#2d1810` (marrom-terroso) → texto, headings, foreground (mantem `--base-dark`)
- Tailwind `base.ts` ja define `base-dark: '#213B2C'` mas globals.css sobrescreve para `#2d1810`
- Decisao: manter ambas como complementares, nao alterar `--base-dark` existente
- [Source: wireframe-home-mobile-first-v3.md#D-1, research#2.3]

### Mandala — Grid Responsivo
- Manter grid 2x2 (D-2), mandala circular e iteracao futura
- Problema atual: `gap: 1.5rem` + `padding: 2rem 1.5rem` SEM breakpoints mobile
- Solucao: 4 breakpoints progressivos no CSS (base, sm, md, lg)
- [Source: wireframe-home-mobile-first-v3.md#secao-3, research#2.2]

### Componentes Existentes para Reutilizar
- `SacredDivider` — usar entre secoes (variantes "line" e "sacred")
- `BaseTriadeFooter` — ja existe em `@ciclo/ui`, mantem
- `BaseTriadeWatermark` — mantem Triskle girando
- `EventCard` — ajustar grid breakpoints (AC-18)
- [Source: visual-spec-v2.md, packages/ui/src/index.ts]

### Conteudo Real Extraido das Referencias
- 6 propositos, 6 facilitadoras com bio, 6 FAQs, 9 atividades timeline, 5 regras cancelamento
- Dados completos no research doc secao 1.3 (Renascenca)
- Contato WhatsApp: Fernanda (47) 99241-4009, Lionara (47) 99758-5250
- [Source: research#1.3, research#3.3]

### Navbar — Especificacao
- Visual Spec v2 secao 3.7 define NavigationHeader mas NUNCA foi implementado
- Wireframe v3 secao 1 detalha: sticky, hamburger mobile, overlay menu, CTA condicional
- Deve ser integrada no layout.tsx (nao no page.tsx) para persistir em todas as paginas
- [Source: visual-spec-v2.md#3.7, wireframe-home-mobile-first-v3.md#secao-1]

### Performance
- NFR-01 do PRD: LCP < 2.5s, FID < 100ms, CLS < 0.1
- Home e Server Component (RSC) — nao precisa de "use client" exceto navbar (useState)
- Navbar pode ser Client Component isolado; resto da page permanece server
- [Source: PRD#NFR-01]
