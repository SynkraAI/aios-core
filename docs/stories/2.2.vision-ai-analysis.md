## Story
**As a** user,
**I want** the app to process my photos with AI, generate a body analysis report and a personalized workout plan, and show me the results,
**so that** I receive accurate scores and actionable guidance about my physique.

## Acceptance Criteria
1. ai-engine (`POST /analyze`) recebe `{ analysis_id, user_id }`, faz download das fotos do S3 via URLs salvas na DB
2. MediaPipe Pose processa as 2 fotos (frente + costas) e extrai landmarks por grupo muscular
3. BodyScores calculados (0–100) para 8 dimensões: `shoulders`, `chest`, `back`, `arms`, `core`, `legs`, `posture_score`, `symmetry_score`
4. Após extração: fotos DELETADAS do S3 imediatamente (LGPD) + `photo_front_url`/`photo_back_url` nullificados + `photos_deleted_at` atualizado
5. Relatório gerado via Claude claude-sonnet-4-6 (prompt caching no system prompt): `highlights` e `development_areas` como array de `ReportSection` (`{ muscle_group, title, description, score }`)
6. Plano de treino gerado via Claude na mesma sessão do pipeline: 4 semanas, 3–5 dias/semana, exercícios detalhados
7. ai-engine chama `POST /internal/analyses/{id}/complete` com `{ scores, report, workout_plan }`
8. API Gateway salva scores em `analyses.scores`, relatório em `reports`, plano em `workout_plans`, marca `status: 'completed'` e `completed_at`
9. Em caso de falha em qualquer etapa: `status: 'failed'`, sem crash — erro registrado no log
10. Loading screen no mobile faz polling `GET /analyses/:id` a cada 2s; em `completed` navega para ReportScreen; em `failed` exibe mensagem com opção de nova tentativa

### Prompt Claude para relatório (com caching)
```python
SYSTEM_PROMPT = """Você é um especialista em avaliação física. Analise os scores de composição corporal e gere um relatório estruturado em JSON com os campos:
- highlights: array de ReportSection com os 3 pontos fortes (score > 65)
- development_areas: array de ReportSection com os 3 grupos mais deficitários (score < 55)

Cada ReportSection tem: muscle_group (string em inglês), title (frase motivacional em PT-BR), description (2-3 frases técnicas em PT-BR), score (inteiro 0-100).
Responda SOMENTE com JSON válido, sem markdown."""

# cache_control no system prompt reduz tokens em ~60% nas re-análises
response = anthropic.messages.create(
    model='claude-sonnet-4-6',
    max_tokens=2048,
    system=[{ "type": "text", "text": SYSTEM_PROMPT, "cache_control": {"type": "ephemeral"} }],
    messages=[{ "role": "user", "content": user_prompt }]
)
```

### Schemas do banco
```sql
-- 003_reports.sql
CREATE TABLE reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  analysis_id UUID NOT NULL REFERENCES analyses(id) ON DELETE CASCADE UNIQUE,
  highlights JSONB NOT NULL,        -- ReportSection[]
  development_areas JSONB NOT NULL, -- ReportSection[]
  generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 004_workout_plans.sql
CREATE TABLE workout_plans (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  analysis_id UUID NOT NULL REFERENCES analyses(id) ON DELETE CASCADE UNIQUE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  duration_weeks SMALLINT NOT NULL CHECK (duration_weeks BETWEEN 4 AND 6),
  sessions_per_week SMALLINT NOT NULL CHECK (sessions_per_week BETWEEN 3 AND 5),
  weeks JSONB NOT NULL,   -- WorkoutWeek[]
  generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```
