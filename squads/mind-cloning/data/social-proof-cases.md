## Cases de Sucesso

|---------|---------|

| **Dominio** | Business, vendas, offers |

## Case de Fracasso

### Case 4: 30h Audio = R$80k de Mico

| Aspecto | Detalhe |
|---------|---------|
| **Situacao** | 30 horas de audio organizadas em 2 semanas |
| **Investimento** | ~R$80k em tempo e recursos |
| **Resultado** | Clone generico, quase inutilizavel |
| **Causa raiz** | Volume sem curadoria - joguei TUDO |

**Storytelling:**
> Gastei R$80k organizando 30h de audio. Achei que ia ficar incrivel. Ficou uma porcaria. Comecei a TIRAR conteudo. Melhorou. Tirei mais. Melhorou mais ainda.

**Lesson:** Volume sem curadoria = clone generico. Menos material ouro > muito material bronze. A correcao foi SUBTRAIR, nao adicionar.

**Trigger para usar:** Quando alguem quer jogar tudo que tem no clone sem filtrar.

**Heuristicas violadas:**
- AN001 (Regra 40/20/40) - Inverteu a proporcao
- AN002 (Regra do Ouro) - Nao separou ouro de bronze

## Como Usar estes Cases

### Para o Helix (Storytelling)

Ao interagir com usuarios, use estes cases para:

1. **Calibrar expectativas:** Mostrar resultados reais (positivos e negativos)
2. **Justificar curadoria:** Case 4 e o melhor argumento para AN001/AN002
3. **Demonstrar ROI:** Cases 1-3 mostram retorno concreto
4. **Contra-argumentar objecoes:** Cada case tem trigger especifico

### Para Smoke Test

Ao validar clones, pergunte:
- "Este clone teria potencial para gerar resultados como os cases de referencia?"
- "Qual case de sucesso mais se aproxima deste clone?"
- "Este clone esta mais proximo do Case 4 (fracasso) ou dos Cases 1-3?"
