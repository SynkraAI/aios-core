# Message Quality Checklist

## Pre-requisitos
- [ ] Dados do prospect carregados (nome, telefone, projeto, score)
- [ ] Regras de mensagem carregadas (data/message-rules.md)
- [ ] Tipo de prospect definido (cliente ou parceiro)

## Humanizacao (CRITICO)
- [ ] Mensagem soa como escrita por pessoa real
- [ ] Nao usa nome + sobrenome junto
- [ ] Nao usa linguagem corporativa
- [ ] Nao parece template
- [ ] Nao tem bullet points ou listas
- [ ] Maximo 5-6 paragrafos curtos
- [ ] Maximo 1-2 emojis

## Personalizacao (CRITICO)
- [ ] Descreve o projeto/negocio especifico do prospect
- [ ] Mostra conhecimento sobre o que a pessoa faz
- [ ] Conecta a dor/necessidade com solucao Ensinio de forma natural
- [ ] Nao e uma mensagem generica que serve pra qualquer um

## Estrutura
- [ ] Saudacao casual com primeiro nome
- [ ] Antonio se apresenta naturalmente
- [ ] Menciona Fosc e o grupo de origem
- [ ] CTA suave e nao agressivo
- [ ] Fechamento caloroso

## Contexto Temporal
- [ ] Se mensagem antiga: contextualiza o timing
- [ ] Se mensagem recente: nao precisa mencionar timing

## Abordagem Correta
- [ ] Cliente potencial: foco em solucao e demo
- [ ] Parceiro: foco em programa de parceiros
- [ ] Nao menciona preco ou planos

## Link WhatsApp
- [ ] Telefone no formato correto (+55XXXXXXXXXXX)
- [ ] Mensagem URL-encoded corretamente
- [ ] Quebras de linha codificadas (%0A)
- [ ] Acentos codificados corretamente
- [ ] Emojis codificados corretamente
- [ ] Link funcional e clicavel

## Aprovacao Final
- [ ] Mensagem revisada contra anti-patterns
- [ ] Passaria no "teste do amigo" (se eu recebesse, acreditaria que e de uma pessoa?)
- [ ] Informacoes do prospect estao corretas
